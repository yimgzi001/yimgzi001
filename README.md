## 简介

我主要从事 Java 后端方向的开发，关注的不只是功能能不能跑起来，也包括系统边界、接口契约、数据一致性、性能和后续维护成本。

目前的技术主线是 Spring Boot 后端开发，同时把 AI 能力作为新的应用层能力来研究：如何把模型调用、知识检索和 Agent 工作流放进一个有边界、有监控、能处理异常的服务里。

## 技术方向

### 后端工程

- Java / Spring Boot / Spring MVC
- REST API、模块化设计与服务边界
- MySQL、Redis、事务与数据访问
- 异步处理、缓存、幂等和任务调度
- 日志、异常处理、测试与可观测性

### Java × AI

- LLM API 集成与流式响应
- Prompt、上下文、会话和 Token 管理
- RAG：文档解析、切分、Embedding、向量检索
- Agent：工具调用、结构化输出和任务编排
- AI 服务的重试、限流、降级与结果校验

### 工程基础设施

- Maven、Git、Linux、Docker
- CI/CD 与环境配置
- API 文档、代码规范和自动化脚本
- 从本地开发到服务部署的完整链路

## 工程关注

我更在意系统长期运行时会遇到的问题，而不只是把功能先做出来：

- 服务边界怎么划分，模块之间怎么协作，才能让代码继续演进？
- 数据一致性、幂等、事务和并发场景，应该怎么取舍？
- 接口出问题时，怎样通过日志、指标和链路信息快速定位？
- 缓存、异步任务和外部依赖出现故障时，系统怎样保持可控？
- AI 能力接入后，如何处理延迟、成本、限流、降级和结果校验？
- 一个原型从能运行到可部署、可测试、可维护，中间还缺什么？
